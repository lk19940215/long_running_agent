# Console CRUD E2E 测试步骤模板

## 任务分解指导
E2E 测试作为独立 test 类任务。steps 用优先级标签标记。

## 标准测试用例

### 【P0】列表加载
1. 导航到列表页
2. Playwright snapshot 验证表格存在
3. 验证至少有 1 行数据（或空状态文案）

### 【P0】搜索功能
1. 在搜索框输入已知关键词
2. 等待列表更新（waitForLoadState）
3. snapshot 验证结果数量变化

### 【P1】分页
1. 点击第 2 页
2. 验证页码高亮变化
3. 验证数据不同于第 1 页

### 【P0】新建
1. 点击新建按钮
2. snapshot 验证弹窗打开
3. 填写表单各字段
4. 提交
5. 验证弹窗关闭 + 列表出现新数据

### 【P1】编辑
1. 点击某行编辑按钮
2. 验证弹窗打开且回填数据正确
3. 修改某字段，提交
4. 验证列表数据更新

### 【P1】删除
1. 点击删除按钮
2. 验证确认弹窗出现
3. 确认删除
4. 验证列表数据减少

### 【P2】表单校验
1. 打开新建弹窗
2. 不填必填项直接提交
3. 验证校验提示出现
4. 填入格式错误的数据（如邮箱格式）
5. 验证格式校验提示
