# API 测试步骤模板

## 标准测试用例

### 【P0】CRUD 正向流程
1. POST 创建资源 → 验证 201 + 返回数据
2. GET 列表 → 验证新数据在列表中
3. GET 详情 → 验证数据正确
4. PUT 更新 → 验证数据变更
5. DELETE 删除 → 验证 GET 返回 404

### 【P0】分页和搜索
1. 创建多条数据
2. GET 列表 ?page=1&pageSize=2 → 验证分页字段正确
3. GET 列表 ?keyword=xxx → 验证过滤结果

### 【P1】参数校验
1. POST 缺少必填字段 → 验证 400
2. POST 字段格式错误 → 验证 400
3. PUT 不存在的 ID → 验证 404

### 【P1】认证鉴权（如适用）
1. 无 token 访问 → 验证 401
2. 错误 token → 验证 401
3. 权限不足 → 验证 403
