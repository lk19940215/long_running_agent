# 03 - 状态管理

> 一句话：一个轻量的 `createStore` 替代 Redux，通过 React Context 注入，`useAppState(selector)` 精确订阅，避免全局重渲染。

## 为什么要单独理解状态管理？

`main.tsx` 花了 2800 行装配出 `initialState`（AppState），然后交给 `launchRepl()` → `App` → React Context → 整个组件树。这个 `AppState` 是**整个应用的状态中枢**，所有模块都通过它通信：

- REPL 读取它来展示 UI
- QueryEngine 通过 `getAppState()` / `setAppState()` 读写它
- 工具执行时通过 `ToolUseContext` 间接访问它
- MCP、插件、Agent 系统都在上面挂载自己的状态

---

## 全景图

```
main.tsx
  ↓ 构建 initialState: AppState
  ↓
launchRepl()
  ↓ App({ initialState })
  ↓   AppStateProvider                    ← React Context Provider
  ↓     createStore(initialState)         ← 创建 Store 实例
  ↓       ↓
  ↓     AppStoreContext.Provider           ← 注入到 React 树
  ↓       ↓
  ↓     REPL + 所有子组件
  ↓       ↓
  ↓     useAppState(selector)             ← 精确订阅某个字段
  ↓     useSetAppState()                  ← 获取 setState 更新器
  ↓
  ↓ 非 React 代码（QueryEngine, 工具）
  ↓   toolUseContext.getAppState()        ← 读状态
  ↓   toolUseContext.setAppState(fn)      ← 写状态
```

## Store 实现 — 35 行替代 Redux

文件：`src/state/store.ts`

```typescript
type Store<T> = {
  getState: () => T
  setState: (updater: (prev: T) => T) => void
  subscribe: (listener: Listener) => () => void
}

function createStore<T>(initialState: T, onChange?: OnChange<T>): Store<T> {
  let state = initialState
  const listeners = new Set<Listener>()

  return {
    getState: () => state,
    setState: (updater) => {
      const prev = state
      const next = updater(prev)
      if (Object.is(next, prev)) return  // ← 引用相等就跳过
      state = next
      onChange?.({ newState: next, oldState: prev })
      for (const listener of listeners) listener()
    },
    subscribe: (listener) => {
      listeners.add(listener)
      return () => listeners.delete(listener)
    },
  }
}
```

**设计特点**：
- **无第三方依赖**：不用 Redux / Zustand / Jotai
- **不可变更新**：`setState(prev => ({ ...prev, field: newValue }))` 模式
- **引用相等优化**：`Object.is(next, prev)` 避免无变化时通知
- **onChange 回调**：Store 创建时可传入变化监听器

## React 集成 — AppState.tsx

文件：`src/state/AppState.tsx`

### Provider

```typescript
function AppStateProvider({ children, initialState, onChangeAppState }) {
  const [store] = useState(() =>
    createStore(initialState ?? getDefaultAppState(), onChangeAppState)
  )
  // 注入到 React 树
  return (
    <AppStoreContext.Provider value={store}>
      {children}
    </AppStoreContext.Provider>
  )
}
```

### 消费方式

| Hook | 作用 | 是否触发重渲染 |
|------|------|--------------|
| `useAppState(selector)` | 订阅 AppState 的某个切片 | ✅ 仅当选中值变化时 |
| `useSetAppState()` | 获取 setState 更新器 | ❌ 稳定引用，永不触发 |
| `useAppStateStore()` | 获取整个 Store（传给非 React 代码） | ❌ 稳定引用 |
| `useAppStateMaybeOutsideOfProvider(selector)` | 安全版，Provider 外返回 undefined | ✅ 同上 |

**关键实现**：`useAppState` 基于 React 18 的 `useSyncExternalStore`，实现精确订阅：

```typescript
function useAppState(selector) {
  const store = useAppStore()
  const get = () => selector(store.getState())
  return useSyncExternalStore(store.subscribe, get, get)
}
```

**使用规范**：
```typescript
// ✅ 选择已有的子对象引用
const verbose = useAppState(s => s.verbose)
const model = useAppState(s => s.mainLoopModel)
const { text, promptId } = useAppState(s => s.promptSuggestion)

// ❌ 不要返回新对象（Object.is 每次都不等，导致无限重渲染）
const bad = useAppState(s => ({ a: s.verbose, b: s.model }))
```

## 状态流转路径

### 从 React 组件更新

```
用户交互（点击/输入/斜杠命令）
  ↓ useSetAppState()
  ↓ store.setState(prev => ({ ...prev, field: newValue }))
  ↓ Object.is 检查
  ↓ 通知所有 listener
  ↓ useSyncExternalStore 触发选中组件重渲染
```

### 从非 React 代码更新（QueryEngine / 工具执行）

```
query.ts / 工具 call()
  ↓ toolUseContext.setAppState(prev => ({ ...prev, ... }))
  ↓ 实际调用的是同一个 store.setState
  ↓ 同样触发 React 组件重渲染
```

### 状态隔离 — 子 Agent

子 Agent 通过 `createSubagentContext` 创建自己的上下文：
- `setAppState` 变为 **no-op**（子 Agent 不能修改父状态）
- `setAppStateForTasks` 保留（可注册/清理会话级基础设施）
- `localDenialTracking` 独立维护（否则拒绝计数永远不累积）

## 设计亮点

| 亮点 | 说明 | 可借鉴场景 |
|------|------|----------|
| **35 行替代 Redux** | 不需要 actions/reducers/middleware，纯函数更新 | 中等规模应用的状态管理 |
| **selector 精确订阅** | `useSyncExternalStore` + selector 避免全局重渲染 | 任何需要高性能状态订阅的 React 应用 |
| **双通道访问** | React（hooks）和非 React（getState/setState）共用一个 Store | 混合架构（React UI + 纯 TS 逻辑）的状态共享 |
| **DeepImmutable 类型约束** | AppState 大部分字段是 `DeepImmutable<...>` | 防止意外直接修改状态 |
| **子 Agent 隔离** | setAppState 在子 Agent 中变为 no-op | 多租户/多代理系统的状态隔离 |
