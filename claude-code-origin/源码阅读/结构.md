# Claude Code 源码架构总览

> 本仓库是 Claude Code v2.1.88 源码泄露后的社区还原版。
> Anthropic 2026年3月在 npm 发布时意外包含了 source map，暴露了 50万+ 行源码。
> Claude Code 并未官方开源，Anthropic 已发起 DMCA 投诉。

## 技术栈

| 层面 | 技术 |
|------|------|
| 运行时 | Bun 1.3.5+（构建/运行/包管理） |
| 语言 | TypeScript (ESM, `.ts` / `.tsx`) |
| UI 框架 | React + 自定义 Ink 终端渲染器 |
| 状态管理 | 轻量 subscribe/getState/setState（非 Redux） |
| Schema 校验 | Zod v4 |
| API SDK | `@anthropic-ai/sdk` |
| 构建宏 | `bun:bundle` 的 `feature()` 条件编译 |
| 插件协议 | MCP (Model Context Protocol) |
| CLI 框架 | Commander.js |

## 核心启动链路

```
bun run dev
  → bootstrap-entry.ts        最小引导：注入全局 MACRO 宏
    → bootstrapMacro.ts        在 globalThis 上挂载 VERSION 等常量
    → entrypoints/cli.tsx      CLI 入口：命令行参数路由分发
      → (11 个快速路径)        --version / daemon / bridge 等，命中即 return
      → main.tsx               默认路径：应用编排中心
        → main()               安全防护 + 协议处理 + 调用 run()
          → run()              Commander 程序创建 + 配置装配
            → preAction()      通用初始化：init() + 迁移 + 远端配置
            → action()         ⭐ 2800 行编排逻辑
              → setup()        设置工作目录 / worktree / 会话 ID
              → 加载命令/工具/Agent
              → showSetupScreens()  信任对话框 / 登录引导
              → launchRepl()   ⭐ 启动交互界面
                → REPL.tsx     交互式主屏
```

## 核心数据流

```
用户输入 → REPL.tsx → QueryEngine.ts → query.ts → Claude API
                                           ↓
                                      工具调用返回
                                           ↓
                                tools/ (执行工具) → 结果回传给 API
                                           ↓
                                query.ts (继续对话或结束)
```

## 六大核心模块

| 模块 | 路径 | 职责 | 规模 |
|------|------|------|------|
| 命令系统 | `commands/` | 87 个斜杠命令（`/help`、`/model`、`/compact` 等） | ~87 目录 |
| 工具系统 | `tools/` | 48 种 AI 工具（BashTool、FileReadTool 等），LLM 调用 | ~48 目录 |
| 终端 UI | `components/` | 终端 UI 组件，基于自定义 Ink 渲染器 | ~400 文件 |
| 服务层 | `services/` | API 客户端、MCP 协议、上下文压缩、遥测 | ~150 文件 |
| 状态管理 | `state/` + `context/` | 应用状态 + React Context | ~15 文件 |
| 终端渲染 | `ink/` | 自定义 React Reconciler → Yoga 布局 → 终端帧输出 | ~100 文件 |
